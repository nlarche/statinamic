During the build process, markdown files are parsed, and transformed as JSON files (one part with the YAML header meta informations, one part with the body as HTML). So both client and server can easily consume markdown files (as json) in order to render pages from React components. Thatâ€™s it !

On top of that you can add pretty much whatever you want to write styles the way you like (CSS, inlines stylesâ€¦) and you can even create your own pages from plain React components. Statinamic is just an helper.

# The boilerplate

  • initialize your bundles ( client + static ) with:
    • your layout components
    • some routes
    • a store to keep data in memory
  • create your build script so you can:
    • define your configuration (eg: webpack loaders to add your favorite css preprocessor)
    • run the dev server ((babel-)node build --server --dev)
    • build the static version for production ((babel-)node build --static --production)

You might as well need to:

  • write or reuse React components
  • tweak webpack configuration in the build script so you can consume images, svg, css or whatever you want.

# Packages used

  • React for writing UI
  • Babel to be able to use ES6/ES2015 today,
  • Webpack for bundling,
  • Redux for handling application state (a better Flux implementation)
